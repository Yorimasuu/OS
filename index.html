Билеты по операционным системам

/01

Что такое LA? В каких единицах измеряется?

LA (Load Average) представляет собой среднюю загрузку системы за определенный период времени. Он измеряется в числе процессов, ожидающих выполнения (в очереди) и может быть представлен в нескольких значениях, обычно за 1, 5 и 15 минут.

/02

Что будет, если на сервере LA = 100?

Когда LA становится очень высоким, это может указывать на перегрузку системы. Значение 100 означает, что в среднем 100 процессов ожидали своей очереди на выполнение. Это может привести к замедлению работы системы, отказам в обслуживании или другим проблемам.

/03

Почему при высоких показателях значения LA на сервере может не наблюдаться проблем?

Высокая загрузка системы не всегда приводит к немедленным проблемам, и это зависит от многих факторов, таких как характер задач, которые выполняет сервер, его общая производительность, доступность ресурсов и т.д. Например, если сервер имеет достаточные ресурсы для обработки всех задач, даже при высокой загрузке, то система может продолжать функционировать нормально.

/05

Как в утилите top в Linux посмотреть нагрузку на каждое ядро процессора?

В top вы можете нажать клавишу '1' (цифра один), чтобы отобразить нагрузку на каждом ядре процессора.

/06

Как в утилите top в Linux посмотреть, какой командой был запущен процесс?

В top нажмите клавишу 'c', и это изменит отображение процессов, чтобы показать полные командные строки.

/07

Где хранятся имена файлов/директорий?

Имена файлов и директорий хранятся в индексе файловой системы, который поддерживает информацию о расположении данных на диске.

/08

Как удалить файл с именем -rf?

Чтобы удалить файл с именем, начинающимся с дефиса, вы можете использовать команду rm -- -rf. Двойной дефис указывает на конец опций команды и гарантирует, что -rf не будет расценено как опция для команды rm.

/09

Как посмотреть описание дескриптора? Как посмотреть время последней модификации файла?

Чтобы посмотреть описание дескриптора, вы можете использовать команду ls -l для отображения подробной информации о файле, включая права доступа, владельца, размер, время последнего изменения и т.д. Для просмотра времени последней модификации используйте ls -l и обратите внимание на столбец "Дата изменения".

/10

Для чего нужна переменная окружения PATH?

Переменная окружения PATH содержит список каталогов, в которых операционная система ищет исполняемые файлы команд. Когда вы вводите команду в командной строке, система просматривает эти каталоги, чтобы найти соответствующий исполняемый файл.

/11

Как посмотреть нагрузку на диски?

Вы можете использовать команду iostat для просмотра статистики использования ввода-вывода, включая нагрузку на диски.

/12

Что такое файл в понятиях Unix-like операционных системах?

В Unix-подобных операционных системах файл — это единица хранения информации, которая может содержать данные, программы или метаданные. Все в Unix-подобных системах, включая устройства и директории, представлены как файлы.

/13

Что такое RAID? Какие массивы бывают?

RAID (Redundant Array of Independent Disks) — это технология, объединяющая несколько физических дисков в один логический массив для повышения производительности, надежности или обоих. RAID 0 (Чередование), RAID 1 (Зеркальный) RAID 5, RAID 6, RAID 10 и модифицированные

/14

При каком количестве одновременно вышедших из строя дисков обеспечивает работоспособность RAID 6?

RAID 6 может выдерживать одновременный выход из строя двух дисков без потери данных. Это достигается за счет использования двух четности (паритетных) блоков, что обеспечивает дополнительный уровень защиты по сравнению с RAID 5.

/15

В чем разница между объявлением переменной export VAR="VALUE" и VAR="VALUE" в bash?

Когда переменная объявляется с использованием export VAR="VALUE", она становится переменной окружения и доступна для всех последующих процессов, запущенных в текущем окружении. Если переменная объявляется просто как VAR="VALUE", она будет видна только в текущем сеансе оболочки.

/16

Как остановить выполнение скрипта в bash при возникновении ошибки в команде?

Для этого используется опция set -e в начале скрипта. Эта опция заставляет скрипт завершаться, если в нем происходит команда с ненулевым кодом возврата.

/17

Что в bash скрипте означает команда set -euo pipefail?

e: Завершать скрипт, если любая команда возвращает ненулевой код возврата. u: Вызывать ошибку при попытке использовать неопределенную переменную. o pipefail: Завершать скрипт, если любая команда в конвейере возвращает ненулевой код возврата.

/18

Как активировать debug режим в bash?

Используйте опцию set -x в начале скрипта, чтобы включить режим отладки.

/19

Что значит $@ в bash?

$@представляет собой все аргументы командной строки, переданные скрипту.

/20

Какой код сигнала будет выполнен при исполнении команды kill?

По умолчанию, если не указан конкретный сигнал, kill отправляет сигнал TERM (15), что означает запрос на завершение процесса.

/21

Как выполнить фильтрацию вывода команды, чтобы на экран были выведены только ошибки (STDERR), игнорируя STDOUT?

Можно использовать конструкцию command 2> error.log, чтобы перенаправить STDERR в файл.

/22

Какую команду необходимо выполнить, чтобы посмотреть, какие пользователи вошли в систему?

Используйте команду who или w, чтобы посмотреть текущих пользователей.

/23

Какой файл необходимо отредактировать, чтобы отключить ssh аутентификацию по паролю?

Необходимо отредактировать файл /etc/ssh/sshd_config и установить параметр PasswordAuthentication no, затем перезапустить службу SSH.

/24

В каком файле находится информация о смонтированных каталогах в файловую систему?

Информация о смонтированных каталогах находится в файле /etc/fstab.

/25

Что выведет команда cat a и почему?

bash mkdir /tmp/abc cd /tmp/abc ls >a 2>b cat a cat a выведет содержимое файла a, в который был перенаправлен вывод команды ls, а STDERR был перенаправлен в файл b( короче, что без цифры или с цифрой 1 в зависимости от порядка получает данные из ls из записывает внутрь, все остальное просто создает файлы).

/26

В bash-скрипте указан атрибут оболочки set -x. В одной из команд происходит ошибка и скрипт завершает свою работу. Как сделать, чтобы при возникновении ошибки в определенной команде скрипт продолжил свою работу?

Можно использовать конструкцию command || true, чтобы продолжить выполнение скрипта даже в случае ошибки.

/27

Что такое системный вызов, какие они бывают?

Системный вызов (System Call) — это механизм взаимодействия программного обеспечения пользователя с ядром операционной системы. Он предоставляет интерфейс для выполнения операций вроде чтения/записи в файл, создания процесса, управления памятью и других базовых функций. Примеры системных вызовов включают open(), read(), write(), fork(), и так далее.

/28

Что такое сигнал в Unix, зачем они нужны и разница между 9 и 15 сигналами?

Сигнал в Unix - это асинхронное уведомление, отправляемое процессу или группе процессов для оповещения о каком-то событии или запросе. Сигналы могут использоваться для управления процессами, обработки ошибок и других сценариев. Разница между сигналами 9 и 15 заключается в их назначении: сигнал 9 (SIGKILL) используется для немедленного завершения процесса и не предоставляет ему возможности выполнить какие-либо дополнительные действия перед завершением, в то время как сигнал 15 (SIGTERM) посылает запрос на завершение, предоставляя процессу возможность закрыть файлы, завершить деятельность и так далее.

/29

Что такое inode? Какая информация там хранится?

Inode (index node) - это структура данных в файловых системах Unix, которая хранит метаданные файла, кроме его имени и фактического содержимого данных. Информация, хранящаяся в inode, включает в себя номера блоков данных, размер файла, время последнего изменения, права доступа и т.д.

/30

Что такое hard link? В чем разница между hard link и soft link? Примеры их практического применения.

Hard link — это дополнительное имя (ссылка) к файлу, которое указывает на тот же inode, что и первоначальное имя. Они не имеют отдельных inode и блоков данных. Soft link (symbolic link) - это отдельный файл, который содержит путь к целевому файлу. Различия: Hard link указывает на тот же inode, soft link - на другой inode, который содержит путь к файлу. Hard link может быть создан только для файлов в пределах одной файловой системы. Применение: Hard links используются для создания дополнительных ссылок на существующие файлы без дублирования данных. Soft links предоставляют более гибкий способ создания ссылок между файлами и каталогами.

/31

Какие состояния процессов существуют? Что значит состояние процесса D?

Состояния процессов включают: Running (R): Процесс выполняется. Sleeping (S): Процесс ожидает возможности выполнения. Stopped (T): Процесс приостановлен. Zombie (Z): Процесс завершил выполнение, но его структура еще не удалена. Состояние процесса D (Uninterruptible Sleep) означает, что процесс ожидает завершения блокирующей операции ввода-вывода и не может быть прерван.

/32

Что такое процесс-зомби и процесс-сирота? Можно ли самостоятельно сделать зомби?

Процесс-зомби - это процесс, который завершил выполнение, но оставил запись в таблице процессов, чтобы родитель мог получить информацию о завершении. Процесс-сирота - это дочерний процесс, у которого умер родитель. Он будет обслуживаться процессом init. Процесс-зомби создается автоматически после завершения дочернего процесса, и его освобождение происходит, когда родительский процесс прочитает статус завершения.

/33

Что такое файловый дескриптор? Какая информация там хранится?

Файловый дескриптор - это целочисленное значение, которое является ссылкой на файл или другий объект, управляемый ядром операционной системы. Он содержит индекс в таблице файлов процесса, где хранятся метаданные о файле, такие как текущая позиция в файле, права доступа и другая информация.

/34

Что такое buffer/cache память? Для чего нужна?

Buffer/cache память в операционной системе используется для временного хранения данных, чтобы ускорить доступ к ним. Buffer относится к временному хранению данных в памяти, прежде чем они будут записаны на диск. Cache используется для хранения копий данных, которые были предварительно прочитаны из диска, чтобы ускорить доступ к ним при последующих обращениях. Оба служат для оптимизации работы с данными и повышения производительности системы.

/35

Порядок загрузки дистрибутива Linux.

Порядок загрузки Linux включает в себя несколько этапов: BIOS/UEFI: Когда компьютер включается, BIOS или UEFI начинают процесс загрузки, выполняя POST (Power-On Self-Test) и определяя устройства загрузки. Bootloader: Загрузчик (например, GRUB) загружается с устройства загрузки (обычно жесткого диска) и предоставляет пользователю выбор ядра (kernel) для загрузки. Ядро (Kernel): Выбранное ядро загружается в память. Ядро инициализирует аппаратное обеспечение, устанавливает драйверы устройств и запускает процесс инициализации. Initramfs (Initial RAM File System): В некоторых случаях, особенно при использовании динамических ядер, используется initramfs — временная корневая файловая система, которая загружается в оперативную память вместе с ядром. Она содержит необходимые драйверы и программы для загрузки основной файловой системы. Init (Инициализация): Init — это первый процесс, который запускается ядром. Он инициализирует систему, запускает различные службы и программы. Runlevel или systemd: В системах с использованием init, runlevel определяет режим работы системы. В systemd используется аналогичный концепт target. Это определяет, какие службы и функции запускаются в данный момент. Login: Наконец, система достигает состояния, где пользователь может войти в систему, вводя свои учетные данные.

/36

Что такое GitFlow?

GitFlow — это модель ветвления для системы контроля версий Git, предложенная Vincent Driessen. Она предоставляет стратегию для эффективной организации работы с ветками в проекте с использованием Git. Основные ветки включают master (основная ветка для стабильных релизов) и develop (в которой разрабатываются новые функции).

/37

Чем merge отличается от rebase?

Merge (слияние): Создает новый коммит, который объединяет изменения из двух веток. Это создает дополнительный коммит, который объединяет изменения из исходной ветки в целевую. Rebase (перебазирование): Переносит изменения из одной ветки в другую, изменяя базовый коммит. В отличие от merge, rebase создает чистую, линейную историю коммитов, но может быть опасен при работе с общими ветками.

/38

Чем tag отличается от branch?

Branch (ветка): Представляет собой отдельную линию разработки. Изменения в ветке могут быть слиты с другими ветками. Tag (метка): Это метка, прикрепленная к определенному коммиту в репозитории. Обычно используется для пометки релизов. В отличие от веток, теги обычно статичны и не изменяются.

/39

В ветке develop есть коммит с изменениями, которые нужно перенести в ветку master. Как это сделать?

Чтобы перенести изменения из ветки develop в ветку master, вы можете выполнить следующие шаги: Переключитесь на ветку master: git checkout master Выполните слияние (merge) или перебазирование (rebase) изменений из ветки develop. Пример для слияния: git merge develop Пример для перебазирования: git rebase develop

/40

Для чего нужна команда git commit --amend?

Команда git commit --amend используется для изменения последнего коммита. Это может быть полезно, если вы хотите внести изменения в сообщение коммита или в сам коммит, который только что был создан. Команда также может быть использована для добавления изменений, которые были забыты в предыдущий коммит.

/41

Что такое Trunk-based development?

Trunk-based development — это методология разработки, при которой все разработка происходит в одной основной ветке (trunk). Это отличается от других методологий, таких как GitFlow, которые предлагают более сложные модели ветвления. Trunk-based development способствует более быстрой интеграции изменений и уменьшению конфликтов при слиянии.

/42

Состояние репозитория ушло на много коммитов вперед. Как откатить весь репозиторий к определенному коммиту?

Используйте команду git reset для отката репозитория к определенному коммиту. Пример: git reset --hard (SHA-коммита) Обратите внимание, что эта команда удаляет все коммиты, идущие после указанного коммита. Бесплатно сохраните изменения в репозитории перед использованием git reset, так как эта команда переписывает историю коммитов, и вы можете потерять данные. Если вы хотите сохранить изменения в удаленном репозитории, после использования git reset, вам может потребоваться использовать git push --force. Однако, будьте осторожны при использовании --force, так как это может повлиять на других участников проекта.

/43

В репозиторий запушен коммит с изменениями в двух файлах. Как откатить изменения этого коммита?

Если коммит еще не был отправлен в удаленный репозиторий, можно использовать команду git reset или git revert: Используя git reset: `git reset HEAD~1` Это удалит последний коммит из локального репозитория. Внимание: изменения будут потеряны, история коммитов будет изменена. Используя git revert: `git revert HEAD` Это создаст новый коммит, отменяющий изменения последнего коммита. История останется неизменной, и новые изменения будут добавлены.

/44

Что такое Docker? В чем отличие контейнера от образа?

Docker - это платформа для разработки, доставки и запуска приложений в контейнерах. Контейнер - это исполняемая единица программного обеспечения, включающая код, его зависимости, библиотеки и другие настройки, обеспечивающие единообразное выполнение приложения в любой среде. Образ - это статический шаблон, который содержит необходимые компоненты для создания контейнера. Контейнер создается из образа во время выполнения.

/45

Какие инструкции есть у Dockerfile?

Dockerfile - это текстовый файл, содержащий инструкции для построения Docker-образа. Некоторые из основных инструкций в Dockerfile включают: FROM: Указывает базовый образ. RUN: Выполняет команду в контейнере. COPY и ADD: Копирует файлы из локальной машины в контейнер. WORKDIR: Устанавливает рабочий каталог для последующих инструкций. EXPOSE: Указывает порты, которые контейнер будет слушать во время выполнения. CMD и ENTRYPOINT: Задают команду, которая будет выполняться при запуске контейнера.

/46

Чем отличается COPY от ADD в Dockerfile?

COPY: Копирует файлы и каталоги из локальной файловой системы в образ Docker. ADD: Похож на COPY, но также позволяет копировать файлы из URL, а также извлекать и автоматически распаковывать tar-архивы. В большинстве случаев рекомендуется использовать COPY, если вам не нужны дополнительные возможности ADD.

/47

Какие есть best practices для написания Dockerfile?

Некоторые best practices включают: Используйте официальные базовые образы. Уменьшайте количество слоев образа. Оптимизируйте порядок инструкций для лучшей кэшируемости. Очищайте ненужные зависимости и временные файлы. Используйте .dockerignore, чтобы исключить ненужные файлы из контекста сборки.

/48

Какие типы сетевых драйверов используются в Docker?

Docker поддерживает различные сетевые драйверы, включая: bridge: Стандартный драйвер, создает внутреннюю сеть между контейнерами. host: Использует сетевые настройки хоста. overlay: Позволяет создавать многозадачные сервисы с распределенной архитектурой. macvlan: Предоставляет каждому контейнеру свой уникальный MAC-адрес, видимый во внешней сети. none: Отключает сетевую конфигурацию для контейнера.

/49

Что такое эфемерные контейнеры?

Эфемерные контейнеры - это контейнеры, которые создаются для выполнения конкретной задачи и уничтожаются после ее завершения. Этот подход широко используется в микросервисной архитектуре, где каждая служба может быть упакована в свой собственный контейнер и эфемерно развернута при необходимости. Это способствует гибкости и масштабируемости инфраструктуры.

/50

Представлен вывод команды free

$ free -m total used free shared buff/cache available Mem: 6930 3598 843 183 2489 2919 Swap: 15999 4 15995 В выводе команды free -m, столбец available отражает количество памяти, которую можно использовать для новых процессов или для кэширования данных.

/51

Почему доступной (available) памяти сейчас 2919, если свободной (free) памяти 843?

Доступная память (available memory) включает в себя как свободную память, так и занятую, которая может быть освобождена при необходимости. Таким образом, доступная память может быть больше, чем свободная память, поскольку она также включает в себя кэшированную или временно используемую память, которая может быть освобождена при необходимости.

/52

Чем отличается CMD от ENTRYPOINT в Dockerfile?

Назначение: ENTRYPOINT используется для определения основной цели контейнера, в то время как CMD обеспечивает поведение по умолчанию, которое может быть переопределено. Переопределение: При запуске контейнера вы можете легко переопределить инструкцию CMD, предоставив дополнительные аргументы командной строки, в то время как инструкцию ENTRYPOINT нельзя переопределить, если она явно не указана с помощью флага -entrypoint. Взаимодействие: Если в Dockerfile указаны и ENTRYPOINT, и CMD, инструкция CMD предоставляет аргументы по умолчанию для инструкции ENTRYPOINT.
